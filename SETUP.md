# RDN Banking Platform - Setup Guide\n\n## Project Overview\n\nRDN Banking Platform is a comprehensive fintech solution with features for deposits, withdrawals, transfers, investments, loans, and currency exchange.\n\n### Modules Completed\n\n**Backend Modules:**\n- ✅ Authentication (JWT-based)\n- ✅ Users Management\n- ✅ Wallet Management\n- ✅ Cards (Virtual & Physical)\n- ✅ Transfers (Local & International)\n- ✅ Loans & Grants\n- ✅ Deposits (Paystack Integration)\n- ✅ Withdrawals\n- ✅ Investments (with ROI calculations)\n- ✅ Currency Exchange\n- ✅ Transactions Tracking\n- ✅ Audit Logging\n- ✅ Admin Panel\n- ✅ KYC Management\n\n**Frontend Stores (Zustand):**\n- ✅ walletStore\n- ✅ notificationStore\n- ✅ transferStore\n- ✅ cardStore\n- ✅ loanStore\n- ✅ settingsStore\n- ✅ depositStore\n- ✅ currencyStore\n- ✅ withdrawalStore\n- ✅ investmentStore\n\n**Frontend Components:**\n- ✅ Dashboard (Header, Stats, Notifications)\n- ✅ Transfers Tab (Form, History, Beneficiaries)\n- ✅ Deposits Tab (Form, History)\n- ✅ Withdrawals Tab (Form, History)\n\n## Prerequisites\n\n- Node.js 18+\n- PostgreSQL 13+\n- npm or yarn\n- Git\n\n## Environment Setup\n\n### 1. Clone Repository\n\n```bash\ngit clone <repository-url>\ncd rdn-banking-platform\n```\n\n### 2. Backend Setup\n\n```bash\ncd backend\n\n# Install dependencies\nnpm install\n\n# Copy environment variables\ncp .env.example .env\n\n# Update .env with your configuration\n# Important: Set PAYSTACK_DEMO_MODE=true for development\n```\n\n### 3. Database Setup\n\n```bash\n# Create database\ncreatdb banking_platform\n\n# Run Prisma migrations\nnpx prisma migrate dev --name init\n\n# Seed data (optional)\nnpx prisma db seed\n```\n\n### 4. Frontend Setup\n\n```bash\ncd ../frontend\n\n# Install dependencies\nnpm install\n\n# Copy environment variables\ncp .env.example .env.local\n\n# Update .env.local with your API URL\n```\n\n## Running the Application\n\n### Development Mode\n\n**Terminal 1 - Backend:**\n```bash\ncd backend\nnpm run dev\n# Server runs on http://localhost:3000\n```\n\n**Terminal 2 - Frontend:**\n```bash\ncd frontend\nnpm start\n# App runs on http://localhost:3001\n```\n\n### Paystack Demo Mode\n\nFor testing Paystack integration without a real account:\n\n1. Ensure `PAYSTACK_DEMO_MODE=true` in backend `.env`\n2. Use these demo endpoints:\n\n**Simulate Successful Payment:**\n```bash\nPOST /api/webhooks/paystack/demo/success\nBody: { \"reference\": \"<deposit_id>\" }\n```\n\n**Simulate Failed Payment:**\n```bash\nPOST /api/webhooks/paystack/demo/failed\nBody: { \"reference\": \"<deposit_id>\" }\n```\n\n**Health Check:**\n```bash\nPOST /api/webhooks/paystack/health\n```\n\n## API Endpoints\n\n### Authentication\n- `POST /api/auth/register` - User registration\n- `POST /api/auth/login` - User login\n- `POST /api/auth/logout` - User logout\n- `POST /api/auth/refresh` - Refresh token\n\n### Wallet\n- `GET /api/wallet` - Get wallet details\n- `GET /api/wallet/balance` - Get current balance\n- `POST /api/wallet/transfer` - Transfer funds\n\n### Deposits\n- `POST /api/deposits/initiate` - Initiate deposit\n- `POST /api/deposits/confirm` - Confirm payment\n- `GET /api/deposits/history` - Get deposit history\n- `GET /api/deposits/:id` - Get deposit details\n\n### Withdrawals\n- `POST /api/withdrawals/initiate` - Request withdrawal\n- `GET /api/withdrawals/history` - Get withdrawal history\n- `POST /api/withdrawals/:id/cancel` - Cancel withdrawal\n\n### Transfers\n- `POST /api/transfers/initiate` - Initiate transfer\n- `GET /api/transfers/history` - Get transfer history\n- `POST /api/transfers/beneficiaries` - Add beneficiary\n- `GET /api/transfers/beneficiaries` - Get beneficiaries\n\n### Investments\n- `GET /api/investments/plans` - Get investment plans\n- `POST /api/investments/create` - Create investment\n- `GET /api/investments/list` - Get user investments\n- `GET /api/investments/summary` - Get investment summary\n- `POST /api/investments/:id/liquidate` - Early liquidation\n\n### Currency Exchange\n- `GET /api/currency/exchange-rate/:from/:to` - Get exchange rate\n- `POST /api/currency/convert` - Convert currency\n- `GET /api/currency/supported` - Get supported currencies\n- `GET /api/currency/history/:from/:to` - Get rate history\n\n### Cards\n- `GET /api/cards` - Get user cards\n- `POST /api/cards/request` - Request new card\n- `POST /api/cards/:id/freeze` - Freeze card\n- `POST /api/cards/:id/unfreeze` - Unfreeze card\n\n## Project Structure\n\n```\nrdn-banking-platform/\n├── backend/\n│   ├── src/\n│   │   ├── modules/\n│   │   │   ├── auth/\n│   │   │   ├── users/\n│   │   │   ├── wallet/\n│   │   │   ├── deposits/\n│   │   │   ├── withdrawals/\n│   │   │   ├── transfers/\n│   │   │   ├── investments/\n│   │   │   ├── loans/\n│   │   │   ├── currency/\n│   │   │   ├── cards/\n│   │   │   ├── payments/\n│   │   │   ├── kyc/\n│   │   │   ├── audit/\n│   │   │   └── admin/\n│   │   ├── prisma/\n│   │   ├── app.module.ts\n│   │   └── main.ts\n│   ├── prisma/\n│   │   └── schema.prisma\n│   ├── .env.example\n│   └── package.json\n├── frontend/\n│   ├── src/\n│   │   ├── stores/\n│   │   │   ├── walletStore.ts\n│   │   │   ├── depositStore.ts\n│   │   │   ├── withdrawalStore.ts\n│   │   │   ├── transferStore.ts\n│   │   │   ├── investmentStore.ts\n│   │   │   └── ...\n│   │   ├── components/\n│   │   │   ├── dashboard/\n│   │   │   │   ├── tabs/\n│   │   │   │   ├── deposits/\n│   │   │   │   ├── withdrawals/\n│   │   │   │   ├── transfers/\n│   │   │   │   └── ...\n│   │   │   └── ui/\n│   │   ├── App.tsx\n│   │   └── main.tsx\n│   ├── .env.example\n│   └── package.json\n└── SETUP.md\n```\n\n## Paystack Integration\n\n### Production Setup\n\n1. Get Paystack credentials from https://dashboard.paystack.co\n2. Update `.env` with real keys:\n   ```\n   PAYSTACK_SECRET_KEY=sk_live_...\n   PAYSTACK_PUBLIC_KEY=pk_live_...\n   PAYSTACK_DEMO_MODE=false\n   ```\n\n3. Configure webhook in Paystack dashboard:\n   - URL: `https://your-domain.com/api/webhooks/paystack`\n   - Events: `charge.success`, `charge.failed`\n\n### Webhook Verification\n\nPaystack will send signed webhooks. The signature is verified using the secret key.\n\n## Testing\n\n### Backend Tests\n```bash\ncd backend\nnpm run test\n```\n\n### Frontend Tests\n```bash\ncd frontend\nnpm run test\n```\n\n## Deployment\n\n### Backend (Docker)\n```bash\ncd backend\ndocker build -t rdn-banking-backend .\ndocker run -p 3000:3000 --env-file .env rdn-banking-backend\n```\n\n### Frontend (Vercel/Netlify)\n```bash\ncd frontend\nnpm run build\n# Deploy the 'dist' folder\n```\n\n## Database Migrations\n\n### Create New Migration\n```bash\nnpx prisma migrate dev --name description_of_change\n```\n\n### Apply Migrations\n```bash\nnpx prisma migrate deploy\n```\n\n## Troubleshooting\n\n### Port Already in Use\n```bash\n# Change port in .env or kill process\nlsof -ti:3000 | xargs kill -9\n```\n\n### Database Connection Error\n- Check DATABASE_URL in .env\n- Ensure PostgreSQL is running\n- Verify database exists\n\n### Paystack Payment Fails\n- Check PAYSTACK_SECRET_KEY is correct\n- Ensure PAYSTACK_DEMO_MODE matches your environment\n- Check webhook logs in Paystack dashboard\n\n## Support\n\nFor issues or questions, please create an issue in the repository.\n\n## License\n\nMIT License - See LICENSE file for details\n"
